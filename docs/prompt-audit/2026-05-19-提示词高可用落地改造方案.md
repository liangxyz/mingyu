# 提示词高可用落地改造方案

生成时间：2026年5月19日

本文替代此前的提示词审计报告和系统补全方案。旧文档偏复盘和补丁记录，本方案只保留可执行、可落地、可验收的工程改动。

本方案目标不是做比赛题适配，也不是改前端样式，而是把当前项目升级成更稳定的“术数资料包 + 提示词证据链”系统。年份、运限和指定应期继续由现有选择器写入提示词；星盘复用同一选择器思路；前端只保留必要的数据入口，不新增视觉样式任务。

## 1. 最终目标

项目最终应达到以下状态：

- 每个体系都有稳定的 `【当前时间】`、`【占卜信息】` 或命盘资料、`【分析对象】`、`【问题】`、`【任务】`、`【输出要求】` 结构。
- 每个关键判断都能在资料包中找到主证、辅证、反证或限制。
- 命盘类涉及年份、月份、日期时，只能使用选择器写入的分析对象，不允许模型凭问题文字自行扩展年份。
- 非命盘占法涉及应期时，必须引用本体系的取时证据，不能用卦名、牌名、签诗意象直接硬换算日期。
- 资料包字段要尽量结构化，优先让模型引用“已算出的证据摘要”，而不是让模型自己从原始数据里猜。
- 所有改动必须有提示词结构测试、目标回归测试和真实样本生成验证。

## 2. 总体架构

后续不要再继续堆长提示词。正确路线是把提示词分成四层。

### 2.1 原始盘面层

职责：保存项目已经算出的原始结果。

示例：

- 八字四柱、大运、流年、流月、流日、十神、旺衰、用忌、神煞。
- 紫微十二宫、星曜、四化、三方四正、大限、流年、流月、流日。
- 星盘本命星体、宫位、相位、行运相位。
- 六爻六亲、世应、动变、伏神、空亡、月日。
- 奇门九宫、门星神干、值符值使、空亡、马星。
- 大六壬四课、三传、天将、天地盘、课体。
- 牌卡、灵签、择日候选日期。

要求：原始盘面层只负责事实，不输出模型解释。

### 2.2 证据摘要层

职责：把原始盘面整理成可引用的主证、辅证、反证、应期候选和权重摘要。

示例字段：

- `主证候选：`
- `辅证候选：`
- `反证限制：`
- `应期候选：`
- `权重口径：`
- `取舍证据：`

要求：这是准确性提升的核心，后续开发优先补这一层。

### 2.3 提示词规则层

职责：告诉模型怎么使用资料，而不是代替算法算命。

统一规则：

- 不得编造资料包没有给出的新盘面事实。
- 允许基于资料包做传统推理，但必须标明证据来源。
- 关键判断必须区分主证、辅证、反证或限制。
- 应期必须说明取自哪一层资料。
- 证据不足时给条件式判断，不给绝对结论。

### 2.4 输出验收层

职责：用测试保证提示词没有退化。

验收方式：

- 提示词结构测试：检查 section、关键字段、顺序和边界规则。
- 算法回归测试：检查新增摘要不会破坏排盘。
- 样本生成测试：固定时间、固定随机种子生成完整提示词。
- 文档检查：真实样本必须包含本轮新增字段。

## 3. 不做的事

为了让改造可控，以下内容不纳入本轮落地：

- 不改前端视觉样式。
- 不新增独立比赛答题模式。
- 不读取或依赖比赛正确答案。
- 不让模型自动从问题文本抽取年份并自行展开运限。
- 不把提示词改成自由发挥的长篇玄学说明。
- 不把所有流派规则一次性塞进提示词。

## 4. P0 改造：统一证据契约

P0 是所有体系共享的基础。先做这一层，再继续补各术数细节。

当前状态：已落地第一版统一证据 helper，文件为 `src/lib/prompt-evidence/types.ts` 和 `src/lib/prompt-evidence/format.ts`，并已由 `tests/prompt-evidence.test.ts` 覆盖去重、空值过滤、权重排序和中文标签输出。八字年限、紫微运限、六爻用神评分、择日禁忌筛查和择日取舍证据已接入统一证据类型；后续新增体系字段时继续优先复用这套 helper。

### 4.1 新增统一证据类型

已新增文件：

- `src/lib/prompt-evidence/types.ts`
- `src/lib/prompt-evidence/format.ts`

已落地类型：

```ts
export type EvidenceLevel = '主证' | '辅证' | '反证' | '限制' | '应期';

export interface PromptEvidenceItem {
  level: EvidenceLevel;
  title: string;
  detail?: string;
  source?: string;
  weight?: number;
  tags?: string[];
}

export interface PromptEvidenceBundle {
  title?: string;
  items: PromptEvidenceItem[];
  emptyText?: string;
}
```

落地要求：

- 不要求一次性替换所有旧字段。
- 先新增类型和格式化 helper。
- 后续每个体系逐步把现有摘要迁入这个格式。

验收测试：

- 新增 `tests/prompt-evidence.test.ts`。
- 检查去重、空值过滤、权重排序、中文标签输出。

### 4.2 统一输出字段格式

建议所有体系最终统一输出以下字段，缺少则不显示。当前版本已先保留各体系更贴近术数语义的稳定字段，后续再逐步映射到统一证据类型：

```text
主证候选：
辅证候选：
反证限制：
应期候选：
取舍口径：
```

当前字段映射：

- 八字 `年限触发摘要` 映射到 `主证候选`、`应期候选` 和 `限制`。
- 紫微 `运限命中摘要` 映射到 `主证候选`、`应期候选` 和 `限制`。
- 六爻 `用神评分表`、`原神忌神仇神` 映射到 `主证候选`、`反证限制`。
- 梅花 `类象权重` 映射到 `主证候选` 和 `取舍口径`。
- 小六壬 `复盘信号` 映射到 `应期候选` 和 `限制`。
- 塔罗 `叙事权重` 映射到 `主证候选`。
- 雷诺曼 `组合权重` 映射到 `主证候选`。
- 灵签 `事项映射` 映射到 `取舍口径`。
- 择日 `禁忌筛查`、`禁忌降级`、`取舍证据` 映射到 `反证限制`、`取舍口径` 和 `限制`。

验收标准：

- 所有非自定义占法提示词至少出现一个主证类字段。
- 涉及应期的体系必须出现应期或时间边界字段。

5. 常见相邻组合优先查词典，例如 `骑士+山` 判断消息受阻、推进延迟，`山+太阳` 判断阻力后转明。
6. 词典未覆盖时才回退到两张牌关键词拼接，避免资料缺口导致提示词中断。

验收标准：

- 每个结论必须写“哪几张牌组合成什么现实事件”。
- 单牌关键词不能单独成为最终结论。

4. 后续逐签补人工签意库。
5. 典故只能映射处境，不直接等同现实结果。

验收标准：

## 8. P4 改造：反推时辰

当前状态：已在 `src/lib/birth-time-reverse/prompts.ts` 落地五段式反推结构，不改前端样式，不新增入口。提示词会固定输出候选时辰、已知线索、线索权重、排除理由和下一轮追问；无线索时明确写“暂无可用线索”，避免模型假装已有事件证据。

要改文件：

- `src/lib/birth-time-reverse/prompts.ts`
- `tests/birth-time-reverse.test.ts`

新增字段：

```text
候选时辰：
已知线索：
线索权重：
排除理由：
下一轮追问：
```

已落地动作：

1. 三柱事实和推理分开。
2. 不直接假定时柱。
3. 每一轮最多问 3 到 5 个能排除或提高候选权重的问题。
4. 用户回答后再更新候选权重。
5. 有补充线索时先做线索权重和排除理由；没有补充线索时只设计第一轮排查问题。
6. 权重只允许使用高、中、低、待确认，不输出伪精确概率。
7. 排除理由必须说明依据来自三柱、已知线索还是后续回答；证据不足时写“暂不排除”。

验收标准：

- 没有补充线索时，不假装已有线索。
- 必须说明哪些线索影响哪些候选时辰。
- 下一轮追问必须控制在 3 到 5 个问题。
- 本轮不得直接定时柱。

## 12. 完成定义

一个体系只有同时满足以下条件，才算补齐：

- 资料包有原始事实。
- 资料包有主证、辅证、反证或限制。
- 有应期边界。
- 自定义问题不会强塞任务模板。
- 普通模式有任务和输出要求。
- 有至少一个结构测试。
- 有至少一个回归测试。
- 真实生成样本能看到新增字段。

截至当前版本，P0、P1、P2、P3、P4 已达到上述完成定义；长期维护项仍按 TODO 表继续拆分执行。
